Representing Positive Integers

As we discussed in the previous section, the electrical circuitry of modern computers is specifically designed to store and manipulate binary data. To better understand how data is represented in the computer we must first define two words that are used to describe binary data, namely, bit and byte.

  • bit - binary digit, a 0 or 1
  • byte - a series of 8 bits, 10010101

A byte is a unit of measurement for binary data. All binary data in the computer are stored as one or more bytes. The largest positive integer that can be represented with one byte is 11111111 (25510). Obviously, computers need to be able to store numbers larger than 255, so computers store integer values in multiple bytes. The table below lists various positive integers and the 4 byte binary number that the computer would use to store the number.

Representing Negative Integers

As you can see, it is not difficult for computers to represent positive integers using binary numbers but what about negative numbers? How do you represent negative values using binary numbers? Aren't negative numbers just numbers with a minus sign in front? Perhaps. That is certainly one valid way to think about them. However, that is not the way computers do it. Not because they can't, it just turns out there is an easier, less complex way. Most computer hardware uses a technique called two's complement.

Representing Real Numbers

In computing, we call all noninteger values real numbers. A real number can be defined as a value with a potential fractional part. That is, real numbers have a whole part and a fractional part, either of which may be zero. For example, some real numbers in base 10 are 104.32, 0.99999999, 357.0, and 3.14159.

*10-3 = 1/1000

In binary, the same rules apply but the base value is 2. Since we are not working in base 10, the decimal point is referred to as a radix point, a term that can be used with any base.

How do we represent a real value in the computer? We store the value as an integer and include information showing where the radix point is. That is, any real value can be described by three properties: the sign (positive or negative); the mantissa, which is made up of the digits in the value with radix point assumed to be to the right; and the exponent, which determines how the radix point is shifted relative to the mantissa. To store a floating-point number on a computer, we can store the three values that define it (sign, mantissa, and exponent). For example, according to one common standard, if we devote 64 bits (8 bytes) to the storage of a floating-point value we use 1 bit for the sign, 11 bits for the exponent, and 52 bits for the mantissa.

Both the mantissa and the exponent are stored as binary numbers. There are several different formats used to store these two numbers in memory. Some of the formats use two's complement form to store one or both of the numbers; other formats do not. This topic, however, is beyond the level of our discussion.
