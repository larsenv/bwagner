The Date class and the GregorianCalendar class used in this problem are both part of Java's java.util package. They are used for storing date and calendar information.

Each InventoryItem has a unique ID, a date of entry into the warehouse, and a base price (not shown), which implements the Comparable interface (larger Date objects are later dates). Refer to the following code segment:

int result = date1.compareTo(date2);

The compareTo method provided by the Comparable interface returns one of three values. The compareTo method returns a negative number if date1 is less than date2, it returns a positive number if date1 is greater than date2, and it returns zero if the two dates are equal.

Some inventory items are perishable and some are not, giving the following class hierarchy.

Items that are not perishable have no discounts. The NonPerishable class is implemented as follows.

public class NonPerishable extends InventoryItem
{
    
public NonPerishable(Date entry, double price)

    {
        super(entry, price);
    }
    
    
public double getPrice()

    {
        return super.getBasePrice();
    }
}

Perishable items have an expiration date. Items already past their expiration date when they enter the warehouse are discounted by 90%; otherwise no discount applies.
