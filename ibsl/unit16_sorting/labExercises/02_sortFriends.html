Directions

In this exercise, you are going to sort an array of objects using the selection sort algorithm. As you know from the unit on searching when you need to compare two objects to determine their ordering (alphabetical, numerical, etc), you cannot use the Java relational operators (<, >, <=, >=, ==, !=). Instead you must use a method named the compareTo method.

int result = object1.
compareTo(object2);



This method compares two objects and returns one of three integer values. It returns a negative number if object1 is less than the object2. It returns a positive number if object1 is greater than object2. And finally, it returns a zero if the two objects are equal. 





It is the responsibility of the designer of a class to implement a compareTo method if they wish for there class to use this method to determine object ordering. In their implementation they must determine what it means for two objects to be equal and also what it means for one object to be less than or greater than another object. 



For example, the String class implements a compareTo method. Two String objects are considered equal if they contain the same characters in the same sequence. A String object is consider to be less than another string if the character of the first string comes first in alphabetical order as compared to the second string. If the first two characters are equal the compareTo method advances to the next two characters in the strings and performs the comparison again. It continues this process until the end of at least one the strings is reached.



Question: Which character is less than the other, "a" or "A"? The answer is "A". The reason is the character "A" has an ordinal value of 65 on the ASCII chart, whereas the character "a" has an ordinal value of 97. 65 is less than 97.

int RESULT = this.LASTNAME.compareTo(FRIEND.LASTNAME)

   return this.FIRSTNAME.compareTo(FRIEND.FIRSTNAME)
