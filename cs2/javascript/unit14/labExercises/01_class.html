A Java Class

In Computer Science 1 you learned that the Java language had a mechanism called a class designed to create objects. Look at the following Person class written in Java.

public class Person
{
    // instance variables
    private String firstName;
    private String lastName;
    
    // constructors
    public Person()   // default constructor
    {
        firstName = "";
        lastName = "";
    }

    public Person(String first, String last)
    {
        firstName = first;
        lastName = last;
    }
    
    // accessor methods
    public String getFirstName()
    {
        return firstName;
    }
    
    public String getLastName()
    {
        return lastName;
    }
    
    // mutator methods
    public void setFirstName(String first)
    {
        firstName = first;
    }
    
    public void setLastName(String last)
    {
        lastName = last;
    }
    
    // toString method
    public String toString()
    {
        return "FirstName = " + firstName + "/n" +
                "LastName  = " + lastName;
    }
}

Recall that following good object-oriented programming methodology a programmer declared a class's instance variables private using the private access modifier there by making them invisible to any code outside the scope of the class. To allow other classes, referred to as clients, the ability to access these variables a programmer had to first create an object using one of the the class's constructors.

Person person = new Person("John", "Wayne");

Using the keyword new this code invokes or calls a constructor to create a Person object and initializes the object's instance variables with starting values. In this case, the firstName variable or field is assigned the name "John" and the lastName field is assigned the name "Wayne".

Once an object was created by a class the client program could then send messages to the object using code like the following:

Person person = new Person("John", "Wayne");   // create object

String first = person.getFirstName();    // call to getFirstName method
String last  = person.getLastName();     // call to getLastName method

System.out.println("First Name = " + first);
System.out.println("Last Name  = " + last);

Because the instance variables of the class were declared private they could not be accessed directly so a client program would use accessor methods like getFirstName and getLastName to see their values.

To modify a Person object's variables, after construction, a client program would use mutator methods like setFirstName or setLastName. 

person.setFirstName("Jimmy");      // call to setFirstName method
person.setLastName("Stewart");     // call to setLastName method

Javascript Class

As you have seen, Java and Javascript have some similarities but mostly they are quite different. Most of the reason for this contrast in styles is due to the fact that Java follows the object-oriented programming model, whereas Javascript follows more of a functional approach. In fact, it was not until the release of its 7th Edition know as ECMAScript 2016 or ES6 that Javascript even had the concept or notation of a class. Being a functional language, by the way this is not a bad thing, everything in the language is either a primitive type or an object. This means that even functions in Javascript are objects. Coming from a C++/Java background this took me a while to wrap my head around. Never the less, here is the Person class written in Javascript.

class Person
{
    // constructor initializes object variables
    constructor()
    {
        // fields or properties
        this.firstName = "John";
        this.lastName = "Wayne";
    }

    // method getFullName returns the person's name firstName lastName format
    getFullName()
    {
        return this.firstName + " " + this.lastName;
    }

    // method getFormalName returns the person's name in lastName, firstName
    getFormalName()
    {
        return this.lastName + ", " + this.firstName;
    }
}

The first thing you will notice is that a Javascript class is not as verbose as a Java class. Following is a list of the differences between a Javascript class and a Java class.

  1. A Javascript class has a constructor, but unlike Java, a Javascript class can only have one constructor and its name must be constructor.

    // constructor initializes object variables
constructor()
{
    // fields or properties
    this.firstName = "John";
    this.lastName = "Wayne";
}

  2. A Javascript class must declare its variables or fields within the constructor and the variables must be preceded by the keyword this.

    // fields or properties

    this
    .firstName = "John";

    this
    .lastName = "Wayne";

    In Java terminology a class's variables were generally referred to as instance variables, however in Javascript the terms field and properties are more commonly used.

    Java also uses the keyword this but it is optional when declaring instance variables. It also must be noted that although they both use the keyword this they do not work exactly the same. We will look more closely at their differences in a future lesson.

  3. A Javascript constructor can assign initial values to fields of a class by passing these values through the constructor's parameter list.

    // constructor initializes object variables
constructor
    (first, last)
      // parameter list
{
 // fields or properties
 this.firstName = first;    // assign value stored in first to firstName
 this.lastName = last;      // assign value stored in last to lastName
}

    The variables declared inside the constructor's ending parentheses are called parameters. Parameters are local variables and therefore do not exist outside of the constructor. Their job is to temporarily store the values passed to the constructor from client code, and then in turn pass those values onto the instance variables of the class. Notice that parameters do not have the keyword var preceding their name.

    Here is an example of client code that creates a Person object and passes initial values to object's fields using the constructor's parameter list.

    var person = new Person("John", "Wayne");  // assigns "John" to firstName field
                                           // assigns "Wayne" to lastName field

    Look at the illustration below. The code after the keyword new is actually a call to the Person class's constructor and the string parameters "John" and "Wayne" are passed two constructor parameter variables firstName and lastName.

  4. Javascript does not use access modifiers. This means there are no public, private, or protected members of a class. Everything in a Javascript class it public. This means that both its variables and methods can be accessed outside of the class through the use of an object and dot notation.

    var person = new Person();
var first = person.firstName;   // accessing Person field using dot notation
var last = person.lastName;
var fullName = person.getFullName();  // accessing Person method

  5. Like Java, a function declared inside a class is called a method. A method is a special type of function that can be invoked or called within other methods in the class or outside the class using an object and dot notation. Notice that even though a method is a function its name is not preceded by the keyword function.

    Suppose a new class method is added to the Person class named addDoctorTitle(). This method adds the title "Dr." to beginning of person's name.

    addDoctorTitle()
{
    return "Dr." + 
    this
    .getFullName();
}

    The method getFullName is used to help define the addDoctorTitle method. Methods can be invoked by other methods within a class but the call must be preceded by the keyword this.

    This is an example of a method being invoked outside of its class using an object and dot notation.

    var person = new Person();  // create Person object
var formalName = person.getFormaName();  // accessing Person method
